# Milestone 2 — Quota Exceeded Behavior Toggles UI

## Scope

Add a "CLIProxy Settings" section to the Provider Management modal that displays two toggle checkboxes for the proxy's quota-exceeded behavior (`switch-project` and `switch-preview-model`). The section only renders when `isCliProxyConfigured()` returns true.

This involves:
- Adding CSS styles for the new `.cliproxy-*` classes following the existing `.fallback-section` pattern
- Adding render/attach/load methods to `ProviderManagementUI.ts` following the `renderFallbackSection()`/`attachFallbackListeners()` pattern
- Wiring toggle changes to `setQuotaSwitchProject()`/`setQuotaSwitchPreviewModel()` with error handling and checkbox revert on failure

## Implementation Notes

- The `renderCliProxySection()` HTML must include a `<span id="cliproxy-models-status"></span>` element — Milestone 3 depends on this to display discovered model count. Include it even though it won't be populated until M3.
- The `loadCliProxySettings()` method should use `fetchQuotaExceededSettings()` (the combined parallel fetch), not individual `getQuotaSwitchProject()` + `getQuotaSwitchPreviewModel()` calls, to minimize round-trips.
- Use `querySelector('#cliproxy-switch-project')` etc. within `this.elements.content` (not `document`) to scope queries to the modal, following the pattern in `attachFallbackListeners()` (line 631–635).
- The `attachCliProxyListeners()` change handlers must be `async` to properly `await` the `set*` calls and catch failures for checkbox revert.
- `loadCliProxySettings()` must check `fetchQuotaExceededSettings()` return for `null` (not just falsy). `null` means proxy unreachable → show notice + keep disabled. Non-null means success → enable checkboxes + set checked state (even if both are `false`).
- Since `loadCliProxySettings()` is fire-and-forget async called from `renderProviderCards()` (not `show()`), it runs on EVERY re-render — including when the user adds/removes models or configures providers. Guard DOM updates with `if (!this.elements.content) return;` at the top and after each `await` to handle: (a) modal closing while fetch is in-flight, (b) `renderProviderCards()` being called again before the previous `loadCliProxySettings()` completes.
- Because `loadCliProxySettings()` is called from `renderProviderCards()` and NOT from `show()`, the `show()` method needs no modifications at all. This is cleaner and ensures CLIProxy settings survive internal re-renders (e.g., adding a model to another provider).
